# Firebase Setup Guide

Step-by-step guide to connect this app to your own Firebase project.

## Prerequisites

- Node.js installed
- Flutter SDK installed
- A Google account

## Step 1: Install Firebase CLI

```bash
npm install -g firebase-tools
```

## Step 2: Login to Firebase

```bash
firebase login
```
This opens a browser for Google authentication.

## Step 3: Install FlutterFire CLI

```bash
dart pub global activate flutterfire_cli
```

## Step 4: Create a Firebase Project

Go to [Firebase Console](https://console.firebase.google.com/) and create a new project. Or use CLI:

```bash
firebase projects:create your-project-name
```

## Step 5: Enable Firebase Services

In the Firebase Console for your project:

1. **Authentication** → Get Started → Enable **Email/Password** provider
2. **Authentication** → Enable **Google** provider (optional)
3. **Cloud Firestore** → Create database → Start in **test mode** → Choose nearest region

## Step 6: Configure Flutter App

From the project root directory:

```bash
flutterfire configure --project=YOUR_PROJECT_ID
```

This generates `lib/firebase_options.dart` with your project credentials. Select the platforms you need (Android, iOS, etc.).

## Step 7: Register SHA-1 (Android)

Get your debug SHA-1:

```bash
keytool -list -v -alias androiddebugkey -keystore ~/.android/debug.keystore -storepass android
```

Register it:

```bash
firebase apps:android:sha:create YOUR_ANDROID_APP_ID YOUR_SHA1 --project YOUR_PROJECT_ID
```

Or add it in Firebase Console → Project Settings → Your App → SHA certificate fingerprints.

## Step 8: Update Cloudinary (Optional)

If you want to use your own Cloudinary account, update `lib/core/constants/cloudinary_config.dart`:

```dart
class CloudinaryConfig {
  static const String cloudName = 'YOUR_CLOUD_NAME';
  static const String apiKey = 'YOUR_API_KEY';
  static const String apiSecret = 'YOUR_API_SECRET';
}
```

## Step 9: Deploy Firestore Rules & Indexes

```bash
firebase deploy --only firestore --project YOUR_PROJECT_ID
```

## Step 10: Run the App

```bash
flutter pub get
flutter run
```

## Step 11: Seed Test Data

On the login screen (debug mode only), tap "Seed Test Data" to populate Firestore with 10 test users.

## Files That Change Per Project

| File | What to change |
|------|---------------|
| `lib/firebase_options.dart` | Auto-generated by `flutterfire configure` |
| `android/app/google-services.json` | Auto-generated by `flutterfire configure` |
| `ios/Runner/GoogleService-Info.plist` | Auto-generated (if iOS configured) |
| `lib/core/constants/cloudinary_config.dart` | Your Cloudinary credentials |
| `firebase.json` | Your project ID (auto-updated) |

## Troubleshooting

- **"Unknown calling package name"** → SHA-1 not registered. Follow Step 7.
- **"PERMISSION_DENIED"** → Deploy Firestore rules. Follow Step 9.
- **"Query requires an index"** → Deploy indexes. Follow Step 9.
- **Google Sign-In not working** → Enable Google provider in Firebase Console + register SHA-1.
